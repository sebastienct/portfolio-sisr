Installation de l'agent sur un poste Linux

Serveur Web Installation de l’agent Zabbix Récupération des sources Dans un terminal wget https://repo.zabbix.com/zabbix/5.4/debian/pool/main/z/zabbix-release/zabbix-release_5.4-1%2Bdebian10_all.deb Installation du paquet Dans un terminal sudo dpkg -i zabbix-release_5.4-1+debian10_all.deb modification des configurations de l’agent Ouvrir le fichier /etc/zabbix/zabbix.agentd.conf en tant que root sudo vim /etc/zabbix/zabbix.agentd.conf modifier les valeurs en ajoutant l’adresse IP du serveur Zabbix Server=SERVER_ZABBIX ServerActive=SERVER_ZABBIX Hostname=NOM_DE_LA_MACHINE recharger le service Agent sudo systemctl restart zabbix-agent.service

AD Domain Service

Dans cette procédure sera décrite comment: Configurer le nom du serveur Ajouter le rôle Active Directory Domain Service Créer une Unité d’organisation (OU) Créer un utilisateur lié à l’OU Configurer le Serveur Après avoir installer un nouveau Windows Server ( ici windows 2019 ), ouvrir le Gestionnaire de serveur. Dans le menu Serveur local, cliquer sur le nom de la machine. Dans la fenêtre qui s’ouvre, cliquer sur modifier, Attribuer un nom à l’ordinateur, ensuite cliquer sur autre et changer le suffixe DNS de l’ordinateur.

Création d'une Politique de mot de passe

Afin de garantir une bonne sécurité de notre infrastructure. Il est recommandé par l’ANSSI de respecter quelques règles dans la politique d’élaboration des mots de passe. Pour rappel un mot de passe fort : est unique et non réutilisable pour une autre utilité ne possède pas de lien avec la personne est composé de 12 caractères MINIMUM est composé de caractères Alphanumérique et de caractère spéciaux est renouvelé régulièrement, 90 jours est un bon compromis Pour plus de détail visité l’ANSSi et plus particulièrement la recommandation
